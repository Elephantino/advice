классы

Классы — это способ установить схему для создания объектов с предопределенными свойствами и методами. Создав класс с определенными свойствами и методами, вы можете позже создать экземпляры объектов этого класса, которые унаследуют все свойства и методы этого класса.


В JavaScript мы можем создавать классы следующим образом:
            
class Alien {
    constructor (name, phrase) {
        this.name = name
        this.phrase = phrase
        this.species = "alien"
    }
    fly = () => console.log("Zzzzzziiiiiinnnnnggggg!!")
    sayPhrase = () => console.log(this.phrase)
}

И затем мы можем создать экземпляр объекта из этого класса следующим образом:


const alien1 = new Alien("Ali", "I'm Ali the alien!")
console.log(alien1.name) // output: "Ali"

Классы используются как способ сделать код более модульным, организованным и понятным и широко используются в ООП-программировании.

Но имейте в виду, что JavaScript на самом деле не поддерживает классы, как другие языки. Ключевое classслово было введено в ES6 как синтаксический сахар, облегчающий такой способ организации кода.

Чтобы визуализировать это, обратите внимание: то же самое, что мы делали ранее, определив class, мы можем сделать, определив функцию и отредактировав прототип следующим образом:


function Alien(name, phrase) {
    this.name = name
    this.phrase = phrase
    this.species = "alien"
}

Alien.prototype.fly = () => console.log("Zzzzzziiiiiinnnnnggggg!!")
Alien.prototype.sayPhrase = () => console.log(this.phrase)

const alien1 = new Alien("Ali", "I'm Ali the alien!")

console.log(alien1.name) // output "Ali"
console.log(alien1.phrase) // output "I'm Ali the alien!"
alien1.fly() // output "Zzzzzziiiiiinnnnnggggg"

Любая функция может быть вызвана как конструктор с ключевым словом, newа свойство прототипа этой функции используется для того, чтобы объект наследовал методы. В JavaScript «класс» используется только концептуально для описания описанной выше практики — технически это просто функции.😑

Хотя это не обязательно имеет большое значение (мы по-прежнему можем прекрасно реализовать ООП и использовать классы, как и в большинстве других языков программирования), важно помнить, что в основе JavaScript лежит наследование прототипов.
